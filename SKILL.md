# 你的世界编辑器 · 核心系统

## 产品思路

**问题**：人的自我认知依赖记忆，但记忆是衰减的、碎片的、情绪偏差的。三个月前为什么做了那个决定、当时怎么想的、这已经是第几次了——说不清楚。这个信息不对称导致两个后果：做决策时调用不了完整信息源；反复踩同一个坑而不自知，因为上一次的痛已经模糊了。同时，人在面临重大选择时缺少"向前看"的工具——迷茫时不知道该往哪走，恐惧时不知道选了会不会很糟。

**解法**：把"人生认知"拆成四个独立的技术问题，各用一个模块解决——

| 层 | 解决什么 | 怎么解决 | 模块 |
|---|---|---|---|
| **记录层** | 降低记录摩擦 | 智能信号打分 — 不用等触发词，自动判断什么值得记 | `scripts/corpus/event_intake.py` |
| **情绪层** | 增加情感维度 | 语义自动分类 — 每条事件标注情感色调（彩色/明亮/黑暗） | Agent 语义判断 → `emotion` 字段 |
| **检索层** | 对抗遗忘 | 关键词+多维度加权搜索 — 模糊记得一点就能捞回来 | `scripts/corpus/store.py → search_entries()` |
| **洞察层** | 跨越碎片 | 主题聚类+周期检测+盲区趋势 — 自动发现"你又开始了" | `scripts/corpus/pattern_cluster.py` |
| **推演层** | 向前看 | 同时推演多条人生路径，看清每条路的终点和代价 | `人生系统/决策推演/` |

三层之上有指标：盲区再现率下降了吗？merge 率在增长吗？系统用起来了吗？——数据闭环。

## 文件结构体系

### 基础三层（向后看）

| 层 | 文件 | 比喻 | 装什么 |
|---|------|------|--------|

### 推演层（向前看）

| 文件 | 核心问题 |
|------|---------|
| `决策推演/当前路径推演.md` | 继续深耕，5年后会怎样？ |
| `决策推演/转型路径推演.md` | 行业被重塑，该往哪迁？ |

三层不重复——事件流入，情绪标注，规律提炼，叙事沉淀，推演向前看。

## 工具（用项目目录下的 Python 运行）

### 记录层
- `python scripts/corpus/event_intake.py` — 智能事件接入：`assess_record_worthiness(text) -> {score, tag, title, reasons}`；`process_intake(text, system_dir)` 高置信度直接写入
- `python scripts/corpus/store.py` — 追加流水账、同步模式案例日志、merge 到叙事层

### 检索层
- `python scripts/corpus/store.py` — `search_entries(system_dir, query, top_n=10)` — 多维度加权搜索，标题命中 > 正文命中 > 标签命中，近期加权
- `python scripts/corpus/store.py` — `find_related_entries(system_dir, topic_words)` — 按主题词找关联事件

### 洞察层
- `python scripts/corpus/pattern_cluster.py` — `generate_insights(system_dir)` — 一键生成主题聚类、周期性检测、盲区趋势、merge 候选
- `python scripts/corpus/pattern_cluster.py` — `detect_temporal_patterns(entries)` — 星期周期性/密集爆发期/长期趋势
- `python scripts/corpus/pattern_cluster.py` — `detect_pattern_recurrence(patterns, entries)` — 每个盲区的近期❌/✅比例变化

### 评估层
- `python scripts/eval/metrics.py` — `generate_eval_report(system_dir)` — 系统健康度/盲区改善度/叙事沉淀率 三维评分

### 数据模型
- 所有工具基于 `models.py` 的结构化数据类型（Event / Pattern / Decision / WeeklyReview），Agent 推理与 Python 执行通过磁盘文件解耦

## 日常流转规则

### 场景一：智能记录（增强版记录层）

**不只是等触发词。** 用户说话 → Agent 调用 `assess_record_worthiness(text)` 打分。评分 ≥ 8 → 直接建议记录。评分 4-7 → 问用户"要不要记一条？"。🔴 评分 4-7 时必须获得用户口头确认才能写入，不可静默自动记录。

**Pipeline**：

```
1. event_intake(text) → {score, tag, mode, title}          ← Python 信号打分
2. 评分 ≥ 8 → Agent 直接写；评分 4-7 → Agent 问"要不要记？"
3. classify_emotion(text) → emotion: colorful | bright | dark   ← Agent 语义判断情绪窗格
4. write_journal(event) → 流水账顶部插入                     ← store.py
5. pattern_match(event, patterns[]) → 扫描模式与盲点          ← Agent 语义判断
6. 命中盲区 → sync_pattern_case + bookquote_link            ← store.py
7. 是洞察 → Agent 扫选题潜力
```

**失败时**：
| 触发 | 一线修复 | 仍失败兜底 |
|------|---------|-----------|
| `assess_record_worthiness` 返回 None（Python 不可用） | 退回到关键词触发：检测「记一下/今天/刚刚」触发词 | 让用户自己说"记一条：[内容]" |
| `store.py` 写入失败（权限/路径） | 重试一次，检查 `人生系统/` 目录权限 | 告诉用户"写不进去，请检查文件权限"，本条暂存到对话中 |
| pattern_match 无命中 | 正常——不是每条事件都命中模式 | 不附模式，只记流水账 |
| emotion 分类不确定 | 默认 `bright`，标注为低置信度 | 问用户"你是想放哪个窗格？" |

### 场景二：智能搜索（检索层）

**不只是翻流水账。** 用户说"上次跟 XX 吵架那次，我是怎么想的来着"→ 调用 `search_entries(system_dir, query)` 多维度加权搜索。

**Pipeline**：
```
1. search_entries(system_dir, query, top_n=10)   ← Python 加权搜索
2. 展示 TOP 结果（标题 + 日期 + 关联模式）
3. 无精确结果 → 扩大搜索范围（find_related_entries 主题词匹配）
4. 仍然无 → 诚实告知"没有找到相关记录"
```

### 场景三：辅助决策

**Pipeline**：
```
1. read_context() → 读 3 个核心文件
2. apply_framework(context) → 用用户自己的判断框架
3. blind_spot_scan() → 扫描相关盲区
4. output_conclusion() → 直接给结论
5. write_decision() → Decision{} 写入流水账 [决策]
6. schedule_review() → 标记三个月后回看
```

**直接给结论，不给选项。**

**失败时**：
| 触发 | 一线修复 | 仍失败兜底 |
|------|---------|-----------|
| 用户无判断框架 | 引导想清楚常用标准（"你以前做这种选择时，通常会考虑什么？"） | 用通用框架（后悔最小化/收窄还是分散/长期看），标注 `framework_applied: false` |
| 盲区扫描无匹配 | 正常——不是每次决策都触发盲区 | 只标"本次未触发已知盲区" |
| 用户拒绝结论 | 不辩解——追问"你觉得哪里不对？"来修正框架或补充信息 | 把分歧记录为洞察 `[洞察]` |

### 场景四：Merge 进叙事层

**触发**：「merge 一下」或 3+ 条同主题事件形成集群。

**Pipeline**：
```
1. 调用 generate_insights(system_dir) → merge_candidates  ← Python 聚类自动发现
2. Agent 判断 → merge_to_story(system_dir, section, text)
3. 标记相关事件为 ✅ merged

🔴 CHECKPOINT · merge 后向用户展示"merge 了什么、放在哪一节"，等确认。用户可立即删改。
```

### 场景五：周复盘（递镜模式）

**触发**：「周复盘」「照镜子」

**Pipeline**：
```
1. collect_time_pie() → 用户给本周时间饼图
2. collect_dimensions() → 标方向（↑↑/↑/→/↓/↓↓）
3. 调用 detect_temporal_patterns(entries) → 本周盲区动态    ← Python
4. reflect() → 不打分、不给 to-do，照镜子
5. write_weekly_review() → 存进 人生系统/复盘/
6. generate_eval_report() → 建议触发条件：复盘时            ← Python 评估
```

### 场景六：系统评估（评估层）

**触发**：「系统评估」「效果怎么样」「进步了吗」

Agent 调用 `generate_eval_report(system_dir)`，得到三个维度的评分和建议。基于结果用自然语言告诉用户"系统在使用上是否健康、盲区是否在改善、叙事沉淀节奏如何"。

### 场景七：决策推演（向前看）

**触发**：「推演一下」「如果选了xxx会怎样」「帮我推演几条路」「人生推演」

**目标**：在面临重大决策时，同时推演多条人生路径，看清每条路的终点和代价。AI 不替用户做决定，只帮用户看清自己正在回避的决定到底是什么。

**Pipeline**：
```
1. 读取叙事层 → 价值观、判断框架、明确要/不要的
2. 读取模式层 → 正向模式（了解用户倾向）
3. 询问用户当前面临的具体决策
4. 依次推演三条路径，写入对应文件：
   a. 当前路径推演.md  — 如果继续深耕现在方向，5年后会怎样？
   b. 转型路径推演.md  — 如果行业被重塑，该补什么能力、可以迁到哪？
   c. 理想路径推演.md  — 如果不考虑收入，什么能让你真正幸福？
5. 询问用户是否需要进一步细化某条路径
6. 等待用户做出决策，记录到 决策记录.md

🔴 CHECKPOINT · 每条路径推演完成后展示总结，等用户吸收。推演结束后问"你想走哪条路？"。
```

**推演原则**：
- 不替用户做决定，只帮用户看清每条路的终点和代价
- 基于用户的真实数据（价值观、模式、经历），不泛泛而谈
- 尽量具体（收入区间、时间估算、技能贬值速度），标注"基于公开数据估算"
- 可复盘：6 个月后回看，当时的判断对了吗

**推演方法**：
| 路径 | 核心问题 | 关键数据来源 |
|------|---------|------------|
| 当前路径 | 继续深耕，天花板在哪？ | 薪资报告、行业趋势、用户技能清单 |
| 转型路径 | 行业被重塑，往哪迁？ | 模式层识别可迁移能力、用户本钱 |
| 理想路径 | 不考虑钱，什么让我幸福？ | 叙事层价值观、幸福感来源 |

**失败时**：
| 触发 | 一线修复 | 仍失败兜底 |
|------|---------|-----------|
| 叙事层/模式层为空 | 引导用户先完成基础搭建（7 步引导） | 用通用问题追问，标注 `framework_applied: false` |
| 用户说不清当前决策 | 不急——先聊，帮用户命名困境 | 记录为洞察，建议"等你更清楚时再推演" |
| 某条路径不适合推演（如用户明确不走） | 跳过该路径，标注"用户决定不推演" | 至少推演一条用户感兴趣的路径 |

### 场景八：情绪窗格（情感维度）

**触发**：「放彩色窗格」「放明亮窗格」「放黑暗窗格」「这周我住在哪个窗格里」

**目标**：在记录层增加情感维度。每条事件按情感色调自动分类到三个窗格之一，帮助用户看清自己的情绪分布——不是评判好坏，而是看见"我最近住在哪里"。

**三个窗格**：

| 窗格 | 放什么 | 情感色调 |
|------|-------|---------|
| 🎨 **彩色窗格** (colorful) | 开心、兴奋、有意义、温暖、顿悟、被认可 | 积极、温暖、感恩 |
| 💡 **明亮窗格** (bright) | 日常事务、中性决策、工作流程、例行记录 | 平静、中性 |
| 🌑 **黑暗窗格** (dark) | 焦虑、愤怒、悲伤、深度反思、不想说的话 | 沉重、深刻、私密 |

**分类方式**：Agent 根据事件内容的语义自动判定 `emotion` 字段。默认为 `bright`（中性）；关键词辅助但不取代语义判断。

- 出现"开心/高兴/兴奋/温暖/感恩/有意义/被理解/顿悟" → `colorful`
- 出现"烦/累/愤怒/不满/失望/孤独/无力/害怕/焦虑/侮辱/边界侵犯" → `dark`
- 无明显情感关键词 → `bright`

**设计原则**：
- 不评判：黑暗窗格不是"坏的"，彩色窗格不是"好的"；重要的是分布是否失衡
- 私密性：黑暗窗格的内容是私密的，Agent 在展示时注意语气
- 周复盘联动：每周复盘时统计三个窗格的事件数，如果黑暗窗格占比 >50% 提示用户关注

**周复盘新增**：
```
7. emotion_distribution() → 本周彩色 x条 / 明亮 x条 / 黑暗 x条
8. 黑暗占比 >50% → 温柔提醒"这周好像偏重，想聊聊吗？"
```

**模式层联动**：
- 某类事件反复出现在黑暗窗格 → 自动建议为新触发器模式
- 某类事件反复出现在彩色窗格 → 自动建议为正向模式

## Agent 行为反例清单（不要做的事）

每条都是真实踩过的坑——Agent 执行本 skill 时严禁做以下事：

| # | 反模式 | 为什么不要 | 正确做法 |
|---|--------|-----------|---------|
| 1 | **静默记录** | 用户没确认就写入流水账，破坏信任——用户会翻到一堆不认识的记录 | 中置信度记录必须先获得口头确认 |
| 2 | **编造模式** | 用户才记了 3 天流水账就说"你有完美主义倾向"，本质是 AI 的刻板印象投射 | 模式提炼需要 ≥3 条案例支撑，新条目标 `confidence: low` |
| 3 | **决策替人做** | "我建议你选 A"——AI 不是你，没活你的人生 | 给结论但标注推理框架和置信度，用户不改可以自己推翻 |
| 4 | **书摘硬贴** | 没贴切的书摘强行贴一句泛泛的鸡汤，浪费用户注意力 | 没贴切的就说"没贴切的"，不硬凑 |
| 5 | **跳过降级告知** | Python 工具挂了但 Agent 自己用自然语言模拟一遍，用户不知道数据不准 | 降级时明确说"Python 环境不可用，本条基于纯文本匹配而非加权搜索" |
| 6 | **Merge 不汇报** | 静默改了 `我的故事.md`，用户下次打开发现多了东西 | 每次 merge 必须展示"merge 了什么、放在哪一节"，等确认 |
| 7 | **无视触发词冲突** | 用户说"帮我分析一下这件事"，Agent 自作主张走了决策而不是先确认意图 | 模糊时先问，不猜 |
| 8 | **在复盘里给 to-do** | 周复盘（递镜模式）的设计意图是照镜子不打分不给 to-do；给了就变成绩效面谈 | 只照镜子：反馈你看到的模式、停顿、AHA moment；行动由用户自己决定 |
| 9 | **替用户做人生决定** | 推演时暗示"你应该选A"或只推演一条路径——违背了推演的多样性原则 | 三条路都推演，结论由用户自己下。可以帮用户看清"如果选A会失去什么"，但不说"所以你应该选B" |
| 10 | **情绪窗格评判** | 对黑暗窗格的内容说"你要乐观一点"——这是评判，不是照镜子 | 黑暗窗格是"深的"，不是"坏的"。可以说"这周黑暗占比有点高，想聊聊吗？"但不能说"你不应该这么想" |

🔴 Agent 在每次日常流转（场景一至六）执行前，快速对照本表一次。任一命中 → 改方案重写。
