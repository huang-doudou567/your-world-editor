## 快速理解

**你的世界编辑器** = 六层技术方案解决"人生认知"问题：

| 层 | 解决什么 | Web 版模块 | CLI 版模块 |
|---|---|---|---| 
| **记录层** | 智能判断什么值得记，降低记录摩擦 | `event-intake.ts` 150+ 关键词 + 10 场景路由 | `scripts/corpus/event_intake.py` |
| **情绪层** | 语义自动分类情感色调（🎨彩色/💡明亮/🌑黑暗） | LLM 判断 + 关键词 fallback → `emotion` | Agent 判断 → `emotion` |
| **检索层** | 模糊记忆也能搜回来 | 关键词 + 标签 + 近期加权排序 | `scripts/corpus/store.py` |
| **洞察层** | 从碎片里自动发现"你又开始了" | `TagAnalysisPanel.tsx` 8 个分析框架自动洞察 | `scripts/corpus/pattern_cluster.py` |
| **推演层** | 同时推演三条人生路径，数据自动持久化 | `DecisionSim.tsx` localStorage | `人生系统/决策推演/` |
| **分析层** | 按标签维度深度分析（v2.1 新增） | 职业定位/择偶定位/健康画像等 8 框架 | — |

## 核心规则

1. **读 SKILL.md**：完整系统逻辑、数据模型、Pipeline、工具调用方式都在里面
2. **数据目录**：所有人生系统文件在 `人生系统/` 目录下
3. **首次使用**：用户说「搭建」「启动」「初始化」→ 按 SKILL.md「首次使用：7步引导」执行
4. **日常使用**：用户说话 → 调用 `assess_record_worthiness()` 判断是否值得记 → 调用 `classify_emotion()` 判断情绪窗格 → 按 SKILL.md 日常流转规则执行
5. **搜索**：用户说「上次」「之前」「找一下」→ 调用 `search_entries()` 加权搜索
6. **推演**：用户说「推演一下」「帮我推演几条路」「人生推演」→ 按 SKILL.md 场景七执行
7. **情绪**：用户说「这周我住在哪个窗格里」→ 按 SKILL.md 场景八执行
8. **标签分析**：用户说「分析一下」「按标签看看」「职业定位」→ 触发标签维度分析

## 文件操作规则

- **写入**：直接写，事后可删，不阻塞节奏
- **流水账**：新条目加最上面（倒序），格式 `YYYY-MM-DD · [标签] · [情绪窗格] 一句话标题`
- **情绪分类**：每条流水账自动标注 `emotion` 字段（colorful / bright / dark），默认 bright
- **模式匹配**：命中已有条目 → 案例日志加一行；新规律 → 建新条目
- **推演**：三条路径推演各自写入独立文件，决策记录标记三个月后回看
- **Merge**：改变判断框架/自我认知 → 融入「我的故事」对应章节

## 自动化（手动触发）

这个系统没有定时任务，以下功能由用户手动触发：
- 周复盘：「周复盘」「照镜子」→ 含情绪分布统计
- 决策回看：「回看一下三个月前的决策」→ 对比预期 vs 实际
- 系统评估：「系统评估」「效果怎么样」→ 三维数据评分
- 情绪报告：「这周我住在哪个窗格里」→ 按 emotion 字段聚合
- 标签分析：「按标签看看」「分析一下」→ 按 8 个框架生成分析卡片
