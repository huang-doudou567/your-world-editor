# 人生系统 · 健康模块

## 管什么

用户健康相关全流程：
- **体检后**：结构化指标、历次对比、标异常项
- **平时**：症状日志
- **看完医生**：就诊归档
- **横向**：健康知识库
- **决策**：要不要做检查/治疗/休息——联动 life-os

### 边界

| skill | 管什么 |
|-------|------|
| life-health | 健康的「事」——指标、就诊、症状、知识 |
| life-os | 健康背后的「我」——「用过劳逃避」「为赚钱牺牲健康」等心理盲点 |

```
/人生系统/健康/
├── 健康档案.md        ← 基础信息（身高/血型/过敏史/家族病史）

└── 体检原件/          ← 原始 PDF/扫描件
```

## 场景一：整理体检报告

**触发**：「我体检完了」「整理体检报告」+ 用户上传 PDF / 贴指标

**Pipeline**：

```
1. receive_report() → PDF / 文本 / 指标列表
2. extract_indicators() → 按系统分类（血液/生化/影像/心电等）
3. compare_history() → 读上次体检记录，做趋势对比（↑/↓/→）
4. flag_abnormal() → 标异常项 ⚠️
5. save_to_record() → 体检记录.md 顶部追加
6. create_followup() → 异常项写进"待跟进"区
7. archive_original() → 体检原件/YYYYMMDD-医院名.pdf
8. check_mental_state() → 用户有抗拒/逃避吗？→ 联动 life-os
```

**输出格式**：

```markdown
## YYYY-MM-DD · [医院名]

### 血液
| 指标 | 本次 | 参考范围 | 上次 | 趋势 |
|------|------|---------|------|------|
| 白细胞 | 6.5 | 4-10 | 5.8 | → |
| 血红蛋白 | ⚠️ 110 | 120-160 | 125 | ↓ |

### 待跟进
- [ ] 血红蛋白偏低 → 建议复查 + 补铁
```

**原则**：
- 数字要准确，不四舍五入
- 趋势用 ↑/↓/→
- 异常项必须 ⚠️
- **不解读病情**（AI 不是医生），只整理事实

## 场景二：症状记录

**触发**：「我哪里不舒服」「记一下症状」

**Pipeline**：

```
1. read_symptom_log()
2. structure_symptom() → 1-2 句话
3. prepend_to_log() → 症状日志.md 顶部
4. check_severity() → 严重症状 → 建议立刻就医
5. prompt_doctor_visit() → 提到要看医生 → 提示整理带去
```

**格式**：
```
YYYY-MM-DD · 部位/类型 · 感受 · 持续多久 · 触发因素
```

**原则**：
- 不诊断
- 追问关键细节：持续多久？触发因素？严重程度？
- **严重症状 → 直接说"建议立刻就医"**

## 场景三：就诊归档

**触发**：「我去看医生了」「整理就诊记录」

**Pipeline**：

```
1. collect_visit_info() → 医院/科/医生/主诉/诊断/医嘱/用药/复查时间
2. structure_record() → 标准化格式
3. prepend_to_record() → 就诊记录.md 顶部
4. sync_medication() → 长期用药 → 健康档案.md
5. schedule_followup() → 复查时间 → 提醒加日历
6. link_symptom() → 症状日志对应条目标 ✅ 已就诊
```

## 场景四：健康知识库

**触发**：「记一下健康知识」「医生教我 XX」

**Pipeline**：

```

```

## 场景五：健康决策（联动 life-os ⭐）

**触发**：「要不要做 XX 检查」「该不该停下来休息」「这个治疗值得吗」

**Pipeline**：

```
1. read_health_data() → 体检 + 就诊 + 健康知识
2. read_life_os_data() → 模式与盲点 + 我的故事
3. apply_framework() → 用户判断框架
4. blind_spot_scan() → 重点扫描：
   - 用过劳逃避情绪
   - 精力分散倾向
   - 健康/数字回避
5. output_conclusion() → 直接给结论
6. write_decision() → life-os 流水账 [决策]
```

**判断框架**：
- 三年后悔法：做和不做哪个会后悔？
- 离健康的自己更近还是更远？
- 是不是「用过劳逃避情绪」在驱动？

**输出**：
```
YYYY-MM-DD · [决策] 健康事件名
选择：做 / 不做
理由：1-2 句
盲区扫描：触发的盲区
回看：YYYY-MM-DD
```

## 回写 life-os 触发标准

**必须回写**：
- 体检异常 + 有后续决策 → `[决策]`
- 就诊后重要建议/诊断 → `[决策]`
- 识别到盲区再现/突破 → `[模式]` ❌/✅
- 第一次主动休息（突破「Yes 惯性」身体版）→ `[模式]` ✅

**不需要回写**：
- 日常症状记录
- 普通保健品选购
- 体检数据存档（无异常）

## 常见健康盲区速查

| 盲区 | 典型表现 |
|------|---------|
| 健康/数字回避 | 不想看体检报告 / 看到红字想拖 |
| 用过劳逃避情绪 | 明知该休息还在接事 |
| 精力分散倾向 | 身体已经累了还在加新事 |
| 主动在意身体 ✅ | 开始记录症状、主动看医生 |

每次健康决策时，**主动扫**这几条。

## 降级处理

| 场景 | 策略 |
|------|------|
| 体检报告不完整 | `confidence: low`，标注缺失项 |
| 没有历史体检数据 | 首次建立基线，标 `degraded: "无历史对比"` |
| 症状描述模糊 | 追问关键细节，不猜测诊断 |
| 涉及专业医学判断 | 明确说"建议咨询医生" |
