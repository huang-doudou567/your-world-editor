# 人生系统 · 财务模块

## 管什么

life-money 管**财务的事**——财务操作知识、报价、接单判断。

**"我和钱的关系"**（赚钱焦虑、不敢要高报价等心理层面）由主 skill `life-os` 管，在用户的 `模式与盲点.md` 里。

> 设计意图：遇到钱、税、报价的问题，很多人第一反应是逃避。这个 skill 就是那个出口——把问题存进来、问出来，而不是绕开。

## 场景一：记财务知识

**触发**：「记进财务 skill」「这个财务的事记一下」

**Pipeline**：

```
1. read_knowledge_base() → 财务知识库.md
2. structure_card() → 标准化知识卡片
3. save_or_update() → 对应主题下（有就补充，没有就新建）
4. identify_gaps() → 没搞懂的丢「待搞清楚」区
5. report() → 口头汇报记了什么
```

**知识卡片格式**：

```markdown
### [主题名]（如：核定征收个体户）
**是什么**：一句话定义
**作用**：对我有什么用
**怎么做**：
1. 步骤一
2. 步骤二
**注意**：
- 注意点
**来源 + 日期**：谁说的 / 哪本书 / 链接 · YYYY-MM-DD
```

**待搞清楚区**：
```markdown
## 待搞清楚的问题
- [ ] YYYY-MM-DD · 问题描述（背景：为什么想搞清楚）
```

## 场景二：财务决策

**触发**：「这个值不值得接」「报价定多少」「这笔钱怎么处理」

**Pipeline**：

```
1. read_money_knowledge() → 有没有相关操作知识
2. read_life_os_patterns() → 扫与「钱」相关的盲区
3. apply_framework() → 用用户判断框架
4. blind_spot_scan() → 重点扫描：
   - 报价偏低不敢要高
   - Yes 惯性 / 高估精力
   - 财务/数字回避
5. output_conclusion() → 直接给结论
6. write_decision() → life-os 流水账 [决策]
7. bookquote_link() → 命中盲区时附书摘
```

**联动 life-os 重点盲区**：
- 💰 **报价偏低不敢要高** → 这笔是不是又在让利？
- ⚡ **Yes 惯性 / 高估精力** → 接这个会不会分散精力？
- 🙈 **财务/数字回避** → 是不是不想算这笔账？

**输出**：
- 直接给结论，不给选项
- 写进 life-os `经历流水账.md`：
```
YYYY-MM-DD · [决策] 财务事件名
选择：做 / 不做
理由：1-2 句（用户判断框架）
盲区扫描：触发的盲区
回看：YYYY-MM-DD
```

## 场景三：查财务知识

**触发**：用户问一个财务/税务问题（「核定个体户是什么」「钱怎么合规转出来」）

**Pipeline**：

```
1. search_knowledge_base() → 有没有相关卡片
2. if found → 直接答 + 指出在哪张卡片
3. if not found → 正常回答 + 自动补一张卡片进知识库
4. disclaimer() → 涉及法规/数字，不确定就说不确定
```

---

## 与 life-os 联动

| 方向 | 什么时候 | 怎么做 |
|------|---------|--------|
| money → os | 财务决策碰心理盲点 | 读 `模式与盲点.md` |
| money → os | 财务突破（如"主动要了高价"）| 写流水账 `[模式]` ✅ |
| os → money | 需要理财书摘 | 从用户书摘层取 |

**原则**：
- 决策一律写进 life-os 流水账
- 财务知识库只存「知识」，不存「决策」

## 降级处理

| 场景 | 策略 |
|------|------|
| 知识库为空 | `degraded: "财务知识库为空"`，正常回答 + 引导记录 |
| 用户没有财务相关盲区记录 | `confidence: low`，标注"未识别到相关盲区" |
| 涉及具体法规数字不确定 | 明确说不确定，建议跟专业人核对 |

## 常见财务盲区速查（事先标好）

| 盲区 | 典型表现 |
|------|---------|
| 报价偏低不敢要高 | 总觉得要多了不好意思 |
| 用赚钱换认可 | 不是为了钱，是为了被认可 |
| 财务/数字回避 | 不想算账、不想看账单 |
| Yes 惯性 | 什么都接，结果精力分散 |

每次财务决策前，**主动扫**这几条。
