# 梦境记录模块 (life-dream)

## 定位
记录梦境内容，提取情绪和符号标签，与近期现实事件关联。不追求"解梦"或玄学解读，而是通过结构化记录发现反复出现的潜意识模式。

## 数据模型

### DreamRecord（梦境记录）
```yaml
id: dream_YYYY-MM-DD_001
timestamp: 2026-06-21T08:30:00+08:00  # 记录时间
dream_time: 2026-06-21T04:00:00       # 做梦时间（大概）
source: 口述

# 结构化内容
scene: 在一个陌生的城市里找路
characters:
  - 角色: 一个穿红衣服的女人
    关系: 不认识
  - 角色: 小时候的自己
    关系: 本人
emotions:
  - 情绪: 焦虑
    强度: 8/10
  - 情绪: 迷茫
    强度: 7/10
symbols:
  - 符号: 迷宫
    上下文: 一直在找出口
  - 符号: 红色的女人
    上下文: 站在远处看我
narrative_arc:
  - 起点: 在陌生城市醒来
  - 转折: 看到红衣服女人，追过去
  - 结局: 女人消失，自己更迷路了

# 与现实的关联
reality_links:
  - 现实事件: 最近在考虑换工作，对未来方向不确定
    关联类型: 情绪映射
    关联强度: 高
  - 现实事件: 上周和母亲通电话，聊到小时候的事
    关联类型: 人物/场景触发
    关联强度: 中

# 模式标记
pattern_tags:
  - 反复出现的符号: 迷宫（第3次）
  - 反复出现的情绪: 焦虑（近5个梦境中出现4次）
  - 与模式与盲点的关联: 命中"面对未知时倾向于逃避而非探索"

grounded: true
confidence: 0.8
```

## 执行流程

### 场景1：记录新梦境

**输入**：用户口述梦境（自由文本）
**输出**：DreamRecord + 与近期现实的关联 + 模式标记

```
步骤1：接收用户口述
  - 输入：自由文本（可能模糊、碎片化、不完整）
  - 不做引导性提问，先完整记录用户说的内容

步骤2：结构化提取
  - 提取：场景、人物、情绪（带强度）、符号（带上下文）、叙事弧线（起点→转折→结局）
  - 如果用户描述模糊：标注"推测"，confidence 降低，不脑补

步骤3：符号与情绪打标
  - 符号：提取反复出现的元素（地点、人物、物品、动作）
  - 情绪：标注情绪类型和强度（1-10）
  - 对比历史梦境：检查是否有重复符号或情绪

步骤4：关联近期现实事件
  - 读取近期流水账（近2周）和模式与盲点
  - 检查梦境中的情绪/人物/场景是否与近期现实事件相关
  - 标注关联类型：情绪映射 / 人物触发 / 场景重现 / 未解决的冲突
  - 标注关联强度：高 / 中 / 低

步骤5：模式标记
  - 如果梦境中的符号/情绪在历史梦境中反复出现（≥3次），标注为"反复出现的符号/情绪"
  - 检查是否命中"模式与盲点"中的已有模式
  - 如果发现新模式（用户未意识到的），标注为"潜在模式，需进一步观察"

步骤6：存档
  - 追加到 dreams.md（按时间倒序）
  - 如有模式更新，同步更新模式与盲点.md
  - 输出 DreamRecord 给用户确认
```

**输出模板**：
```markdown
## 梦境记录 - 2026-06-21

**场景**：在一个陌生的城市里找路
**人物**：红衣服女人（陌生）、小时候的自己
**情绪**：焦虑(8/10)、迷茫(7/10)
**符号**：迷宫（第3次出现）、红衣服女人
**叙事**：在陌生城市醒来 → 追红衣服女人 → 女人消失，更迷路

**与现实的关联**：
- 最近在考虑换工作，对未来方向不确定 → 情绪映射（高）
- 上周和母亲通电话，聊到小时候的事 → 人物触发（中）

**模式标记**：
- 反复出现的符号：迷宫（第3次）
- 与"模式与盲点"关联：命中"面对未知时倾向于逃避而非探索"

**原始口述**：
[用户原话]
```

### 场景2：梦境模式汇总（月度）

**输入**：近30天的梦境记录
**输出**：梦境模式汇总报告

```
步骤1：读取近30天的梦境记录

步骤2：统计
  - 出现频率最高的符号（Top 5）
  - 出现频率最高的情绪（Top 5）
  - 与现实事件的关联类型分布

步骤3：模式识别
  - 反复出现的符号组合（如"迷宫 + 焦虑"）
  - 与现实事件的强关联模式（如"每次工作压力大时就会梦到迷路"）

步骤4：生成汇总报告
  - 统计 + 模式 + 建议（如有）
  - 如发现新模式，提议写入"模式与盲点"
```

**输出模板**：
```markdown
## 梦境模式汇总 - 2026年6月

**统计**：
- 本月记录梦境：8个
- 高频符号：迷宫(5次)、水(3次)、追赶(3次)
- 高频情绪：焦虑(6次)、无力感(4次)

**模式识别**：
- "迷宫 + 焦虑"组合出现4次，均与工作压力相关
- 每次梦见"水"时，次日都会感到情绪低落

**与现实关联**：
- 80%的梦境与现实中的未解决冲突相关
- 最强的关联：工作压力 → 迷路类梦境

**潜在新模式**：
- 建议写入"模式与盲点"：工作压力大时，潜意识倾向于用"迷路"表达无力感

**建议**：
- 无明显建议（仅观察和记录）
```

## 与其他模块的关系

### 与主系统 (life-os) 的关系
- **梦境记录 → 流水账**：梦境本身不写入流水账，但梦境中发现的"潜在模式"可以写入"模式与盲点"
- **流水账 → 梦境关联**：梦境记录时会读取近期流水账，检查与现实的关联
- **模式与盲点 ↔ 梦境**：双向关联
  - 梦境可以命中已有模式（强化证据）
  - 梦境可以发现新模式（提议写入模式与盲点）

### 与子模块的关系
- **不直接关联其他子模块**：梦境是独立的记录维度
- **但梦境内容可能涉及其他模块**：
  - 如果梦境涉及工作/求职 → 可能在 life-jobhunt 中有对应事件
  - 如果梦境涉及财务压力 → 可能在 life-money 中有对应事件
  - 通过"模式与盲点"间接关联

## 降级处理
- **用户描述模糊/不完整**：标注"推测"，confidence 降低，不脑补细节
- **无法关联现实事件**：标注"无明显关联"，不强行关联
- **历史梦境数据不足**：无法判断"反复出现的符号"时，标注"需进一步观察"
- **所有降级标记必须在输出中明确告诉用户**
