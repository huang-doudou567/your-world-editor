# 人生系统 · 求职模块

## 管什么

用户作为**候选人**面试的全流程：
- 面试**前**：调研目标公司
- 面试**后**：整理结构化纪要
- 横向：沉淀行业 insight
- 决策：offer 要不要接（联动 life-os）

> 本 skill 是候选人找工作，不是面试官招人。

---

## 场景一：面试前调研

**触发**：「我下周面试 XX」「帮我调研 XX 公司」

**Pipeline**：

```
1. collect_hr_info() → 岗位/面试官/时间/薪资/产品线
2. web_search_company() → 成立时间/融资/产品/创始人/动态
3. competitor_analysis() → 赛道位置 + 对手对比
4. generate_report() → 三段式报告
5. save_report() → [公司名]-面试调研报告.md
6. update_tracker() → _求职追踪表 状态「🔍 调研中」
```

**报告结构**：

```markdown
# [公司名] 面试调研报告
> 面试岗位：XX | 面试时间：YYYY-MM-DD

## 第一部分 · 纵向：公司怎么走到今天的
（按时间线：创立→融资→关键产品→当前阶段）

## 第二部分 · 横向：和谁在打
（赛道位置 + 主要对手对比表）

## 第三部分 · 横纵交汇：现在是什么样的公司
（这个岗位在里面是什么位置、能发挥什么）

## 附录：可能被问到的问题 + 你的钩子
```

**原则**：
- 事实标注来源，推测标"推测"
- 目标是"坐到面试桌上能聊到对方眼睛亮一下"，不是百度百科

## 场景二：面试纪要

**触发**：「我面完了」「整理面试笔记」

**Pipeline**：

```
1. collect_info() → 公司/岗位/时间/面试官
2. structure_notes() → 标准化纪要
3. save_notes() → YYYYMMDD-面试纪要-[公司名].md
4. auto_extract_insight() → 触发场景三
5. update_tracker() → 触发场景四
6. run_life_os_framework() → 📌 当前倾向（联动核心）
```

**纪要结构**：

```markdown
---
tags: [面试纪要, 公司名, 岗位]
date: YYYY-MM-DD
面试对象: 公司名
岗位: 岗位名
面试官: 姓名/职位
当前倾向: 待观察 / 想接 / 不接
---

# [公司名] 面试纪要 · YYYY-MM-DD

## 🎯 一句话判断
（回看时能立刻想起值不值得的总结）

## 一、公司 & 团队
## 二、产品 & 战略
## 三、岗位职责
## 四、薪资 & 福利
## 五、面试官观点 / insight

## ✅ 吸引我的点
## ❌ 风险 / 疑点

## 下次要问的问题
- [ ] 关键合同条款
- [ ] 产品真实数据
- [ ] 试用期目标

## 📌 当前倾向（联动 life-os ⭐）

**倾向**：待观察 / 想接 / 不接

### 三年后悔法
- 一年/两年/三年后接和不接分别会怎样

### 收窄还是分散？
- 对照用户 `我的故事.md` 里的方向

### 盲区扫描
- 对照用户 `模式与盲点.md`
- ⚠️ 标注每一条可能触发的盲区

### 综合判断
- 下一轮必须拿到什么答案
- 拒绝盲签的底线
```

**关键**：`📌 当前倾向` 必须真的去读用户的 `模式与盲点.md` + `我的故事.md`，跑一遍 life-os 框架。不是写"待观察"敷衍。

**整理原则**：
- 保留用户原话味道
- 数字要准确
- 模糊的标"待确认"

## 场景三：沉淀行业 insight

**触发**：自动跟在场景二之后；也可独立触发「沉淀这条 insight」

**Pipeline**：

```
1. extract_insight(notes) → 非公司专属、可复用的认知
2. format_insight() → 标准化格式
3. save_insight() → 行业insight.md 顶部追加
```

**提取标准**：
- 团队建设思路、产品策略、商业判断、行业现象
- 问"6 个月后回看还有用吗？"——只留有用的

**格式**：
```markdown
### YYYY-MM-DD · [来源：公司名/赛道] · [主题]
一段话总结 insight。
> "面试官原话" / "我的总结"
```

## 场景五：Offer 决策（联动 life-os ⭐）

**触发**：「这个 offer 要不要接」「在 A 和 B 之间选」

**Pipeline**：

```
1. read_jobhunt_data() → 追踪表 + 纪要 + 调研 + insight
2. read_life_os_data() → 我的故事（判断框架）+ 模式与盲点
3. apply_framework() → 用用户的框架分析
4. blind_spot_scan() → 扫描相关盲区
5. output_conclusion() → 直接给结论
6. write_decision() → life-os 流水账 [决策]
7. update_tracker() → 状态改为 ✅已接 / ❌我拒
8. schedule_review() → 三个月后回看提醒
```

**判断框架**（来自 life-os）：
- 三年后悔法：一年后/两年后/三年后，哪个会后悔？
- 收窄还是分散：加深当下方向还是分散精力？
- 盲区扫描：是不是"不敢要高价"让用户低估了薪资？是不是"Yes 惯性"让用户仓促接？

**输出**：
- 直接给结论，不给选项
- 写入 life-os 流水账：
```
YYYY-MM-DD · [决策] [公司] offer
选择：接 / 不接
理由：1-2 句（用户判断框架）
盲区扫描：触发的盲区
回看：YYYY-MM-DD
```

## 与 life-os 联动

| 方向 | 什么时候 | 怎么做 |
|------|---------|--------|
| jobhunt → os | 重要决策 | 写回流水账 `[决策]` |
| jobhunt → os | 命中盲区 | 同步模式与盲点案例日志 |
| jobhunt → os | 认知突破 | 追加流水账 `[洞察]` |
| os → jobhunt | 决策时 | 读模式与盲点 + 我的故事 |

## 降级处理

| 场景 | 策略 |
|------|------|
| 用户没投喂过简历/资料 | `degraded: "缺少候选人画像"`, 提醒补充 |
| `我的故事.md` 无判断框架 | `framework_applied: false`, 引导建立 |
| 网络搜索失败 | 标注信息来源缺失，建议用户补充 |
| 追踪表为空 | 首次触发自动建空模板 |
